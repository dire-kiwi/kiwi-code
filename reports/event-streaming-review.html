Findings

HIGH

Committed transitions could disappear for slower clients

The subscriber channel contained only one empty wake-up. If another mutation occurred before a client handler consumed it, the second signal was dropped. When the handler eventually woke, it read the latest live state, not the state associated with the original transition. That violates the requested “all status events to all clients” behavior.

MEDIUM

Global status was split across two long-lived connections

Each browser opened one SSE connection for project/thread snapshots and another for Pi activity. The split duplicated reconnect lifecycle and consumed two persistent HTTP connections per UI even though both streams had the same global audience.

COVERAGE

Tests did not model the failure mode

The project stream test used two clients, but consumed every snapshot before making the next mutation. Pi activity had only a single stream reader. No automated test attached multiple clients to one tmux session while keeping an unrelated session open.

VERIFIED

Terminal bytes were already scoped correctly

Every terminal WebSocket created its own PTY-backed tmux attach-session process. The tmux session name included project, thread, and tool IDs, and the PTY read loop wrote only to that WebSocket. There was no global terminal broadcaster leaking bytes to unrelated clients.

Concrete loss scenario

T0Pi finishes

A “changed” token is queued for clients A and B.

T1Client A reads quickly

A reads current state, sees finished, and acknowledges it.

T2State is cleared

The tracker deletes the activity. B’s channel is still full, so the new wake-up is dropped.

T3Client B finally reads

B consumes its old token but reads the new empty state. It never receives the finished transition.

Recommended remediation

Capture an immutable snapshot at mutation time, queue snapshots in order for each subscriber, expose one named global SSE endpoint for browser state, retain terminal data on session-scoped WebSockets, and add an executable headless multi-client contract test.

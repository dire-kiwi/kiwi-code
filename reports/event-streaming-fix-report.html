What changed

01

Lossy signals became a bounded broker

internal/broadcast owns subscriber registration and immutable queues. Project and Pi publishers enqueue every snapshot in order; a stalled subscriber is disconnected at a fixed backlog instead of blocking writes or consuming unbounded memory.

02

Independent domains share one connection

GET /api/events sends named projects and pi-activity messages. Keeping the payloads separate avoids cross-domain rollback under bursts. Project events refresh external Git status and Pi events filter removed threads.

03

Terminal lifecycle is explicit

Plain HTTP probes cannot start tmux, WebSocket handshakes precede session creation, ping/pong deadlines remove dead peers, and deleting a project stops every thread/tool session before removing its record.

04

Reusable headless contract runner

internal/headless and cmd/headless-client health-check the server, coordinate simultaneous clients, test heartbeats and transitions, verify tmux fan-out/isolation/closure, support remote paths, and offer -skip-terminal.

Compatibility and operating boundary

The project-only and Pi-only SSE endpoints are retained for compatibility, but the UI uses the named global stream. Connected consumers receive ordered snapshots; a consumer that exceeds 256 queued snapshots is deliberately disconnected so EventSource can reconnect to authoritative initial state. The stream is state delivery rather than a durable audit log, so it does not replay intermediate events missed while offline. Terminal window selection remains tmux session semantics.
